Role Capabilities
Level 1

Super Admin

Platform owner · Full access
  • Create, edit & delete agencies
  • Manage all flights, Umrah & Hajj
  • Define custom package types (UAE Visa, UK Visa…) & assign to agencies
  • View & confirm all bookings
  • Manage banks & payment accounts
  • Broadcast announcements to all L2 & L3
  • View all ledger & reports
  • Upload agency logos (binary in DB)
Level 2

Master Agent

Multi-agency manager
  • Create & edit agencies (no delete)
  • Manage flights, Umrah & Hajj packages
  • Approve/reject sub-agency-admin bookings
  • Manage payment accounts
  • Broadcast announcements to direct children
  • Receives super_admin announcements
  • View ledger & reports
Level 3

Agency Admin

Single-agency manager
  • Manage own agency settings & logo
  • Manage flights, Umrah & Hajj
  • Manage granted custom package types (dynamic sidebar items)
  • Assign custom packages to specific sub-agents
  • Own bookings auto-confirmed instantly ⚡
  • Approve/reject sub-agent bookings
  • Manage payment accounts for agents
  • Broadcast announcements to sub-agents
  • Receives from super_admin & master
Level 4

Sub Agent

Booking agent
  • View flights, Umrah, Hajj & assigned custom packages
  • Create bookings (stays PENDING)
  • Submit payment slips (binary image)
  • View own bookings & ledger
  • Request cancellation of confirmed bookings
  • Receives announcements from agency admin
  • Cannot confirm any booking

Key Business Rules

Booking Rules

  • Sub Agent (L4) cannot confirm any booking
  • Agency Admin (L3) bookings are auto-confirmed instantly
  • No user can confirm their own booking
  • Approval queue shows only direct children's PENDING
  • Booking expires after agency-configured minutes (default 60)
  • Confirmation triggers PDF invoice + ledger DEBIT/CREDIT

Announcement Rules

  • Sub Agent (L4) cannot create announcements
  • super_admin targets broadcast to all Level 2 & Level 3
  • CHILDREN targets go to direct parentId matches only
  • Date filter: validFrom ≤ today ≤ validUntil
  • Bell count tracked via localStorage.readAnnouncements
  • Opening an announcement decrements bell count

Image Storage (Binary DB)

  • All images stored as BYTEA in PostgreSQL — no filesystem
  • Announcement images → announcements.image_data
  • Payment slips → booking_payments.slip_image_data
  • Agency logos → agencies.logo_data
  • Content-type column stored alongside each binary column
  • Served via GET /api/.../image endpoints (auth-gated)

Permissions Model

  • super_admin uses role-based permissions (all 18)
  • All other roles use per-user custom permissions
  • Custom permissions allow fine-grained user-level overrides
  • Hibernate agencyFilter scopes data by agencyId
  • Native SQL used where agencyFilter must be bypassed (parentId lookup)
  • UserTypeLevel (1–4) drives auto-confirm & announcement logic

Custom Package System

  • Super Admin defines Package Types (e.g. UAE Visa, UK Package)
  • Types are assigned per-agency via agency_package_grants
  • Agency Admin sees dynamic sidebar items for each granted type
  • Agency Admin creates packages under a type, optionally restricts to specific sub-agents
  • visible_to_all=true → all agency sub-agents see it
  • visible_to_all=false → only listed users in custom_package_user_grants
  • Booking uses bookableType=custom with quota enforcement
  • Quota full → 409 CONFLICT returned; UI disables button proactively
